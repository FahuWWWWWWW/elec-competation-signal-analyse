## 方案A：纯MCU方案（STM32H743VGT6）

### 1. 系统架构

```
信号输入 (50mV~10Vpp, 1kHz~100kHz)
    │
    ├── [输入保护]  TVS+串联电阻+钳位二极管
    │
    ├── [程控衰减/放大]  PGA AD8251 (增益×1/×2/×4/×8)  
    │                    前端电阻网络衰减(×0.1/×0.2/×1)
    │
    ├── [抗混叠LPF]  3阶巴特沃斯 LPF, fc = 500kHz
    │
    ├── [ADC驱动]  OPA2170 差分驱动
    │
    ├── [STM32H743 内置ADC1]  16bit / 3.6MSPS max
    │
    ├── [DMA]  双缓冲循环采集 (每帧N=4096点)
    │
    ├── [MCU处理]  FFT(Flat-top窗) → 谐波提取 → THD计算
    │
    └── [输出]  TFT-LCD显示波形/频谱 + UART蓝牙至手机
```

### 2. 前端调理电路

**2.1 输入保护**
- TVS: PESD5V0S1UB (双向5V钳位，响应<1ns)
- 串联限流电阻: 1kΩ
- 钳位二极管: BAT54S (肖特基，Vf≈0.3V) 至 ±3.3V

**2.2 程控增益级**
- PGA: AD8251 (单通道，增益×1/×2/×4/×8，带宽10MHz@G=8)
- 前端电阻分压网络（继电器切换，用于10V量程）:
  - 最大输入保护：10Vpp → 先衰减至 ≤3.3Vpp
  - 档位设计:
    | 输入范围 | 前端衰减 | PGA增益 | 等效总增益 | ADC输入 |
    |----------|---------|---------|-----------|---------|
    | 50mV~200mV | ×1 | ×8 | ×8 | 400mV~1.6V |
    | 200mV~1V | ×1 | ×2 | ×2 | 400mV~2V |
    | 1V~3.3V | ×1 | ×1 | ×1 | 1V~3.3V |
    | 3.3V~10V | ×0.33 | ×1 | ×0.33 | 1.1V~3.3V |

**2.3 抗混叠滤波器**
- 类型: 3阶巴特沃斯 LPF（单位增益Sallen-Key拓扑 + 1阶RC）
- 截止频率: fc = 500kHz（保护500kHz内谐波分量, 抑制>500kHz噪声）
- 设计计算（取C1=C2=100pF）:
  - 2阶节: R1 = R2 = 1/(2π·fc·C·√(Av)) → 设Av=1
  - R = 1/(2π × 500kHz × 100pF) ≈ 3.18kΩ, 取3.3kΩ
  - 1阶RC节: R = 3.3kΩ, C = 100pF → f = 482kHz
  - 实测-3dB ≈ 490kHz
- 运放: OPA2170 (轨到轨, 带宽3MHz, 适用于音频段)

### 3. 后处理与算法

**3.1 ADC配置**
- ADC1: 16bit分辨率, 采样率 fs = 2MSPS
- 时钟: 来自定时器TIM2触发, 时钟分配使ADC时钟=20MHz
- 采样周期: 10.5个ADC时钟周期（≈525ns）
- 转换周期: 8.5个ADC时钟周期
- 总周期: 19个ADC时钟 → 单次转换时间 = 19/20MHz = 0.95μs → 等效1.05MSPS
- 双ADC交替模式可提升至2.1MSPS
- DMA: 双缓冲各4096点, 满帧触发中断

**3.2 采样参数**
- 采样点数: N = 4096
- 采样率: fs = 2MSPS
- 频率分辨率: Δf = fs/N = 2000000/4096 ≈ 488.28Hz
- 针对100kHz基频: 每周采样点数 = 2000000/100000 = 20点（满足FFT要求）
- 频率分辨率488Hz可分辨100kHz和100.488kHz，足以分离各次谐波

**3.3 FFT与窗函数**
- 窗函数: Flat-top窗（余弦窗族，精确幅值测量）
  - 5项系数: a0=0.21558, a1=0.41663, a2=0.27726, a3=0.08375, a4=0.00693
  - 幅值精确度: <0.01dB（无需频域插值即可直接读幅值）
  - 主瓣宽度: 约3.77个bin → 对基频和谐波无混叠影响（488Hz×3.77≈1.84kHz，远小于100kHz基频间距）
  - 旁瓣抑制: -93dB
- FFT: 4096点实数FFT (Cortex-M7 DSP指令, 约36万时钟周期≈0.75ms@480MHz)
- 峰值搜索: 基频至5次谐波附近各搜索±50个bin内的峰值

**3.4 THD计算**
- THD = √(U₂² + U₃² + U₄² + U₅²) / U₁ × 100%
- U₁~U₅: 各次谐波经Flat-top窗修正后的幅值
- 频域搜索范围: 基频f₀ ± 1kHz, 2f₀ ± 2kHz, ..., 5f₀ ± 5kHz

**3.5 输出/通信**
- TFT-LCD 3.5寸 320×480, SPI接口, 显示时域波形+频谱柱状图
- 蓝牙: HC-05/HC-06 通过UART传输数据至手机APP

### 4. 关键参数验证

**验证1：ADC采样率满足5次谐波Nyquist条件**

> 最高谐波频率: f_max = 5 × 100kHz = 500kHz  
> Nyquist采样率要求: fs ≥ 2 × f_max = 1MSPS  
> 实际ADC配置: fs = 2MSPS  
> 满足系数: K = 2MSPS / 1MSPS = 2.0, 满足Nyquist且留有1倍余量

**验证2：频率分辨率与谐波分离**

> 基频f₁ = 100kHz, 2次谐波间隔 = 100kHz  
> Δf = fs/N = 2×10⁶/4096 = 488.28Hz  
> 100kHz / 488.28Hz ≈ 205个bin → 谐波间隔至少100个bin以上, 完全可分离  
> Flat-top窗主瓣宽度3.77个bin < 谐波间隔, 无频谱混叠

**验证3：THD测量误差理论分析**

> ADC量化误差(16bit): SNR = 6.02×16 + 1.76 = 98.08dB  
> 考虑Flat-top窗处理增益 ≈ -5.5dB(幅值修正后)  
> 有效SNR ≈ 98.08 - 5.5 ≈ 92.6dB  
> 可测最小谐波分量 ≈ -92.6dBc, 对应THD ≈ 0.0023%  
> 远低于3%目标误差 → 量化噪声可忽略  
> 主要误差源: 前端模拟电路非线性失真(Typ < 0.01%) + 基频极小幅值抖动

**验证4：实时性**

> 4096点采集时间: T_acq = 4096 / 2MSPS = 2.048ms  
> 4096点FFT运算时间(Cortex-M7 @480MHz): ≈ 360000 / 480MHz ≈ 0.75ms  
> THD计算/峰值搜索: ≈ 0.1ms  
> 总处理时间 ≈ 3ms (单帧)  
> 满足≤10秒要求

### 5. 优缺点

**优点:**
- 单芯片方案，BOM简单，布板面积小
- H743内置ADC性能优秀(16bit/3.6MSPS)，无需外部ADC
- Cortex-M7 DSP指令加速FFT，满足实时性
- 功耗低（~200mW）

**缺点:**
- 内置ADC的INL/DNL不如专用ADC (典型INL ±3LSB)
- 同步采集通道数有限（仅1路高速）
- 动态范围受限于3.3V电源轨
- 模拟前端未集成，需外部PGA+LPF

## 方案B：MCU+专用ADC方案（STM32H723ZGT6 + AD7980）

### 1. 系统架构

```
信号输入 (50mV~10Vpp, 1kHz~100kHz)
    │
    ├── [输入保护]  TVS+钳位
    │
    ├── [程控衰减]  电阻网络+继电器
    │
    ├── [差分PGA]  THS4521 全差分放大器 (Gain可调)
    │
    ├── [3阶抗混叠LPF]  fc = 500kHz
    │
    ├── [ADC驱动器]  ADA4940-1 单端转差分
    │
    ├── [ADC]  AD7980  16bit/1MSPS SAR, SPI输出
    │            REF: ADR4525 (2.5V, 0.02%)
    │
    ├── [MCU: STM32H723ZGT6]
    │       SPI+DMA接收ADC数据
    │       FFT(Flat-top窗) + THD计算
    │
    └── [输出]  TFT-LCD + 蓝牙/手机
```

### 2. 前端调理电路

**2.1 输入保护与衰减**
- 同方案A，但增加前端精密电阻分压网络
- 衰减挡: ×1, ×0.2, ×0.1（由松下AQW214光继电器切换）

**2.2 差分PGA**
- THS4521: 全差分运放，带宽145MHz@G=1
- 增益设置: 外部电阻网络 (RF/RG)
  - G=1: RF=499Ω, RG=499Ω（差分输入阻抗=1kΩ）
  - G=2: RF=499Ω, RG=249Ω
  - G=5: RF=499Ω, RG=100Ω
- 差分输出可有效抑制共模干扰 + 提高SNR ~3dB

**2.3 抗混叠LPF**
- 类型: 3阶巴特沃斯，差分结构（两个2阶Sallen-Key交叉耦合 + 1阶RC）
- fc = 500kHz, 设计同方案A
- 运放: OPA2196 (高精度, 轨到轨, 2.5MHz)

**2.4 ADC驱动**
- ADA4940-1: 低功耗差分ADC驱动器
- 固定G=1, 输出共模=2.5V (VREF/2)
- 差分输出到AD7980差分输入
- RC抗混叠: R=20Ω, C=1nF (f-3dB≈8MHz)

**2.5 专用ADC: AD7980**
- 分辨率: 16bit, 无失码
- 采样率: 1MSPS
- INL: ±0.5LSB typ, ±1.5LSB max
- SNR: 91dB typ @ 10kHz input
- 动态范围: 96dB
- 电源: 2.5V单电源, 功耗≈15mW@1MSPS
- 参考源: ADR4525 (2.5V, 0.02%, 2ppm/°C)
- 接口: SPI (4线, 最高50MHz, 转换时间约710ns)

### 3. 后处理与算法

**3.1 ADC配置**
- 采样率: fs = 1MSPS (CNV=1MHz脉冲)
- 由STM32H723定时器产生CNV脉冲
- SPI时钟: 50MHz, 单次读取16bit需320ns
- 转换时间: 710ns (tCONV)
- 周期: tCNV = 1μs, 转换+读取 ≈ 710+320 = 1030ns, 满足1MSPS

**3.2 算法流程**
- 采样点数: N = 2048 (1MSPS下, Δf=488.28Hz, 同方案A)
- FFT: 2048点复数FFT, STM32H723 Cortex-M7 @550MHz
- Flat-top窗加权
- 峰值检测: 抛物线插值提高频率精度
- THD计算: 同方案A

**3.3 校准**
- 直流偏置校准: 无信号输入时记录ADC偏置，软件扣除
- 增益校准: 输入精确1Vpp正弦，软件校准增益系数

### 4. 关键参数验证

**验证1：ADC有效位数（ENOB）**

> AD7980 @ 10kHz SNRtyp = 91dB  
> ENOB = (SNR - 1.76) / 6.02 = (91 - 1.76) / 6.02 = 14.82 bit  
> 加上ADA4940-1 (SNR≈105dB@10kHz)，系统SNR ≈ 1/(1/10^(91/10) + 1/10^(105/10))  
> ≈ 1/(1/1.26e9 + 1/3.16e10) ≈ 1.21e9 → SNR ≈ 90.8dB  
> ENOB ≈ (90.8-1.76)/6.02 = 14.79bit

**验证2：THD误差预算**

> ADC THD: AD7980典型-110dB @ 10kHz  
> 前端运放THD: OPA2196典型-120dB @ 10kHz  
> PGA THD: THS4521典型-115dB @ 10kHz  
> 系统总THD ≈ -110dB (最差) → 对应0.0003%  
> 相比3%要求预留约100倍余量  
> 主要误差源由算法和Flat-top窗幅值还原精度决定

**验证3：整周期采样适配**

> 对于100kHz基频: 周期T = 10μs  
> M = fs / f₁ = 1000000 / 100000 = 10点/周期  
> N = 2048 = 204.8个周期 → 非整数周期  
> Flat-top窗抑制频谱泄漏: 旁瓣-93dB足够

### 5. 优缺点

**优点:**
- 专用ADC性能远超内置ADC (INL±0.5LSB, SNR 91dB)
- AD7980的16bit精度保证低失真测量
- 差分信号链抗干扰能力更强
- 分离式设计便于调试和信号链优化

**缺点:**
- BOM成本高于方案A
- PCB面积增大（多芯片）
- SPI接口限制了ADC数据速率（1MSPS下SPI需50MHz）
- 需额外参考源ADR4525

## 方案C：FPGA异构方案（Artix-7 XC7A35T + STM32F407VGT6）

### 1. 系统架构

```
信号输入 (50mV~10Vpp, 1kHz~100kHz)
    │
    ├── [输入保护]  TVS+钳位
    │
    ├── [PGA]  AD8251 (×1/×2/×4/×8)
    │
    ├── [抗混叠LPF]  4阶椭圆LPF, fc=500kHz (椭圆滤波过渡带更陡)
    │
    ├── [ADC]  AD9226 (12bit, 65MSPS) — 高速采集
    │
    ├── ┌──────────────────────────────────────────┐
    │    │  Artix-7 XC7A35T FPGA                       │
    │    │   • 双通道DMA数据采集控制                    │
    │    │   • 4096点流水线FFT引擎 (Flat-top窗)          │
    │    │   • 谐波峰值搜索 (状态机式)                  │
    │    │   • SPI/UART桥接 → STM32                    │
    │    └──────────────────────────────────────────┘
    │
    ├── ┌──────────────────────────────────────────┐
    │    │  STM32F407VGT6                             │
    │    │   • 人机交互 (按键+LCD菜单)                │
    │    │   • THD最终计算 + 归一化处理               │
    │    │   • 蓝牙通信 → 手机显示                   │
    │    │   • FPGA配置加载 (SPI Slave)               │
    │    └──────────────────────────────────────────┘
    │
    └── [输出]  TFT-LCD + 蓝牙
```

### 2. 前端调理电路

**2.1 PGA**
- AD8251: 同方案A, 增益×1/×2/×4/×8
- 增加前端衰减: ×0.1 (用于10Vpp输入)

**2.2 抗混叠滤波器**
- 类型: 4阶椭圆LPF (Cauer滤波器, 过渡带最陡)
- 截止频率: fc = 500kHz
- 通带纹波: 0.1dB
- 阻带衰减: 60dB @ 800kHz
- 拓扑: 2阶Sallen-Key级联 (2个2阶节)
- 运放: OPA2822 (双通道, 高速, 80MHz GBW)
- 元件: 查椭圆滤波器归一化表

| 节 | R1 | R2 | C1 | C2 |
|----|----|----|----|----|
| 1 | 1.2kΩ | 4.7kΩ | 220pF | 56pF |
| 2 | 1.5kΩ | 3.3kΩ | 180pF | 47pF |

**2.3 ADC: AD9226**
- 分辨率: 12bit
- 最大采样率: 65MSPS
- 输入范围: 1Vpp ~ 5Vpp (可通过VREF调节)
- 本设计配置: fs = 20MSPS, 输入范围2Vpp
- SNR: 70dB @ 5MHz input
- SFDR: 82dBc @ 5MHz input
- 接口: 12bit并行CMOS输出 (采样率20MSPS时完全可行)

### 3. 后处理与算法

**3.1 FPGA数据采集控制**
- AD9226在20MSPS下持续采集
- 采样深度: 4096点 → 实际耗时 4096/20MHz = 204.8μs
- 数据通过FIFO缓存后写入Block RAM
- 配置采样率可调: 1MSPS / 5MSPS / 10MSPS / 20MSPS (适应不同基频)

**3.2 FPGA FFT引擎**
- Xilinx FFT IP核: 4096点, 16bit定点, 流水线架构
- 流模式: 约4096+FFT延迟周期≈5000时钟@20MHz=250μs
- 窗函数: Flat-top窗系数存入ROM, 相乘后送入FFT核
- 峰值搜索: 状态机在FFT完成后扫描频谱,提取各谐波幅值

**3.3 STM32F407处理**
- 通过SPI从FPGA接收: 基频+2~5次谐波幅值 (共5个16bit值)
- THD最终计算: THD = √(ΣAᵢ²)/A₁ × 100%
- 显示控制: TFT-LCD
- 蓝牙通信: HC-05通过USART发送测量结果

**3.4 硬件加速优势**
- 采集 + FFT在FPGA内完成: <500μs
- 相比MCU纯软件: 加速比 ≈ 5~10倍 (4096点FFT)
- 可连续采集多帧做平均降噪

### 4. 关键参数验证

**验证1：过采样提升有效位数**

> AD9226: 12bit @ 20MSPS, SNR = 70dB  
> 本设计实际需求: fs_req = 1MSPS (Nyquist对500kHz)  
> 过采样率: OSR = 20MSPS / (2×500kHz) = 20  
> 理论SNR提升: ΔSNR = 10·log₁₀(OSR) = 10·log₁₀(20) ≈ 13dB  
> 等效SNR ≈ 70 + 13 = 83dB → ENOB ≈ (83-1.76)/6.02 ≈ 13.5bit  
> 超过12bit要求

**验证2：FFT实时性**

> FFT时钟: 20MHz  
> FFT处理时间: 4096/20MHz ≈ 205μs (流模式)  
> 数据采集: 204.8μs  
> 总采集+处理: ~410μs  
> 10秒内可处理: 10000000 / 410 ≈ 24000帧  
> 即使做100帧平均, 也仅需41ms, 实时性极高

**验证3：ADC SFDR对THD测量影响**

> AD9226 SFDR = 82dBc @ 5MHz  
> 最差情况谐波分量 = -82dBc (相对于基波)  
> 对应可测THD下限 ≈ 10^(-82/20) × 100% ≈ 0.008%  
> 远满足3%要求

### 5. 优缺点

**优点:**
- FPGA流水线FFT速度极快 (微秒级完成)
- 20MSPS高速采样支持过采样提升SNR
- 可扩展更多功能（实时频谱显示、波形记录）
- AD9226并行接口简单, 时序易满足

**缺点:**
- FPGA开发难度高, 调试复杂
- BOM成本最高 (FPGA ¥60-80)
- PCB要求高 (4层以上, BGA封装)
- 功耗较大 (~500mW)
- 12bit ADC量化精度不如方案B的16bit

## 三方案对比总结

| 维度 | 方案A (纯MCU) | 方案B (MCU+专用ADC) | 方案C (FPGA异构) |
|------|-------------|-------------------|----------------|
| ADC性能 | 16bit/3.6MSPS, INL±3LSB | 16bit/1MSPS, INL±0.5LSB | 12bit(+过采样→13.5bit)/20MSPS |
| 系统SNR | ~85dB | ~91dB | ~83dB (过采样后) |
| FFT速度 | ~0.75ms | ~0.5ms | ~0.2ms |
| 开发难度 | ★★ (低) | ★★★ (中) | ★★★★★ (高) |
| BOM成本 | ~¥157 | ~¥284 | ~¥315 |
| 适用场景 | 快速原型/基础题 | 高精度/发挥部分 | 高速/实时频谱/扩展 |
| THD达标 | 基本满足 | 最优余量 | 基本满足 |

**推荐**: 若追求平衡推荐方案A (H743) 作为主方案, 方案B作为高精度备选。方案C适合有FPGA开发经验的队伍冲刺极致性能。
