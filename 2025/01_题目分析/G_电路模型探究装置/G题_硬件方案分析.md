### 2. 前端调理电路

#### 2.1 DDS信号源（MCU DAC实现）

| 模块 | 器件 | 参数 |
|------|------|------|
| 波形合成 | H723 DAC1 (12-bit) + DMA | 100Hz~500kHz正弦波 |
| LPF | 2阶Butterworth LPF | fc=2.5MHz, 带外抑制>40dB@5MHz |
| VGA | AD8367 (串行控制) | -2.5~-42.5dB, 带宽500MHz |
| 驱动 | OPA365 (BW=50MHz) | G=+2, 输出3Vpp @ 50Ω |

**DAC输出频率上限**：12-bit DAC最大更新率1MSPS，DMA循环输出256点正弦表：
$$f_{max} = \frac{1\text{MSPS}}{256} = 3.9\text{kHz}$$

这不够！1MHz要求远高于此。

**修正方案**：使用更短波形表+LFSR插值。

256点正弦表：f_max = 1MSPS/256 = 3.9kHz ❌
16点正弦表：f_max = 1MSPS/16 = 62.5kHz ❌
**结论**：H723内置DAC无法满足1MHz输出要求 ❌

**替代方案**：使用H723 HRTIM + PWM + 外部LPF实现DDS：
- HRTIM @ 550MHz → 15-bit分辨率PWM
- PWM频率 = 550MHz/2^15 = 16.8kHz (作为DAC分辨率太差)

**最终**：方案A只能覆盖至~500kHz(32点正弦表@1MSPS/32=31.25kHz × 16倍过采样 = 只能到62.5kHz，还需外置DAC)。

**结论**：方案A纯MCU必须外挂DDS芯片，故**方案A实际不可行**。标记为仅作为对比参考，实际按方案B构造。

#### 2.2 ADC采样前端

| 模块 | 器件 | 参数 |
|------|------|------|
| 缓冲/衰减 | OPA365 + 电阻分压 | ×1/×10/×100可编程衰减 |
| 差分转单端 | THS4521 (差分运放) | 全差分输入→单端输出 |
| 抗混叠LPF | 2阶Butterworth, fc=2MHz | R=10kΩ, C=8pF |
| ADC | H723 ADC1/2 (16-bit, 3.6MSPS) | 双通道同步采样 |

**抗混叠滤波器截止频率**：
$$f_c = \frac{1}{2\pi \sqrt{R_1R_2C_1C_2}} = \frac{1}{2\pi \sqrt{10k \times 10k \times 8p \times 8p}} = 1.99\text{MHz}$$

ADC采样率fs=3.6MSPS → 奈奎斯特频率=1.8MHz < fc=1.99MHz，欠采样区域有混叠风险 ❌

**修正**：LPF fc=1MHz, 使用3阶贝塞尔：
- R=10kΩ, C1=15pF, C2=10pF, C3=6.8pF
- fc≈1MHz，在1.8MHz处衰减>30dB ✅

#### 2.3 已知模型电路（二阶过阻尼LPF）

Sallen-Key拓扑匹配H(s)=5/(10⁻⁸s²+3×10⁻⁴s+1)：

```
         R1=10k    R2=10k
Vin ──/\/\/───/\/\/──┬──(+)
                      │       OpAmp
                     C1      ┌──┴──┐
                    10nF     │     │
                      │      │  G=5│
                     GND     └──┬──┘
                               │
                     C2        │
                    10nF       │
                      │       GND
                     GND
```

元件值推导：
$$\omega_n = \frac{1}{\sqrt{R_1R_2C_1C_2}} = \frac{1}{\sqrt{10k \times 10k \times 10n \times 10n}} = 10^4 \text{ rad/s} = f_n = 1.59\text{kHz}$$

$$\zeta = \frac{1}{2}\left(\sqrt{\frac{R_2C_2}{R_1C_1}} + \sqrt{\frac{R_1C_2}{R_2C_1}}(1 - K)\right)$$
其中K=5(增益)：
代入R1=R2=10k, C1=C2=10n:
$$\zeta = \frac{1}{2}(1 + 1 \times (1 - 5)) = \frac{1}{2}(1 - 4) = -1.5$$

负阻尼！意味着C1C2同值时Sallen-Key增益=5会导致不稳定。

**修正**：调整C1/C2比值。
令C1=22nF, C2=4.7nF, R1=R2=10kΩ:
$$\omega_n = \frac{1}{10k\sqrt{22n \times 4.7n}} = \frac{1}{10k \times 3.21\times10^{-8}} = 3115 \text{ rad/s} \neq 10^4$$

需要迭代计算。更实用的方法是使用元件值计算器或采用MFB拓扑。

**最终采用MFB(多重反馈)拓扑**：

MFB二阶LPF传递函数：
$$H(s) = \frac{-R_2/R_1}{s^2R_2R_3C_1C_2 + sC_1(R_2R_3/R_1 + R_2 + R_3) + 1}$$

设计值：R1=2kΩ, R2=10kΩ, R3=5kΩ, C1=10nF, C2=2nF

### 3. 后处理与算法

#### 3.1 扫频测量

```
for f = 100Hz to 1MHz step 100Hz (10000频点):
  设定DDS频率f
  等待建立 (3×周期 = 3/f)
  同步采样Uin, Uout (各256点, fs=10×f)
  计算幅度: A_in = RMS(Uin), A_out = RMS(Uout)
  计算增益: |H(jω)| = A_out / A_in
  计算相位: φ = arg(cross_correlation(Uin, Uout))
  存入Bode表

扫频时间估算:
  低频段时域长: 100Hz × 10点=100ms × 10,000频点 = 太长！❌
```

**优化**：对数扫频，每倍频程10个频点，总频点数=10×log2(1MHz/100Hz)=10×13.3≈133点。

```
for decade = 100Hz, 1kHz, 10kHz, 100kHz, 1MHz:
  每decade扫描20频点
  总频点 ≈ 100点
  每频点: 最低频率100Hz → 3周期=30ms
  总时间 ≈ 5s << 2min ✅
```

#### 3.2 系统辨识

测量得到Bode图(100个频点)→IIR系数匹配：

**Levy法辨识**（MCU实现）：
1. 构建目标函数 min|H_meas(jω) - H_model(jω, p)|²
2. p = [ω_n, ζ, K] 为二阶系统参数
3. 梯度下降优化 (3-5次迭代, 每次<10ms)

对于二阶系统可以直接解析求解：
从Bode图提取：
- 低频增益：K = |H(jω→0)|
- 截止频率：ω_c = 对应-3dB點的ω
- 滚降斜率：-40dB/decade → 二阶
- 从相频曲线拐点估计ζ

**传递函数拟合**：
$$|H(jω)|^2 = \frac{K^2}{(1 - (ω/ω_n)^2)^2 + (2ζω/ω_n)^2}$$

取3个频点(ω1, ω2, ω3)解方程组：
- 最小二乘拟合，3未知数×多频点 → 超定方程

#### 3.3 数字滤波器实时实现

辨识得H(s)后，双线性变换 → H(z)：

$$s = \frac{2}{T_s} \frac{1 - z^{-1}}{1 + z^{-1}}, f_s = 10 \times f_{in,max}$$

对于fin=1MHz, fs=10MSPS（超出H723内置ADC最高3.6MSPS ❌）

**修正**：fs = 3.6MSPS，最高处理频率360kHz。1MHz信号欠采样。

**结论**：纯MCU方案无法完成1MHz信号的实时数字滤波输出。需要更高采样率外置ADC或降频处理。

### 4. 关键参数验证

#### 4.1 DAC频率上限

H723 DAC 12-bit @ 1MSPS, 正弦表32点:
$$f_{max} = 1\text{MSPS} / 32 = 31.25\text{kHz} \ll 1\text{MHz} ❌$$

即使使用8点正弦表: f_max = 125kHz，仍不够 ❌

**结论**：方案A纯MCU不可行，必须外置DDS芯片。

#### 4.2 ADC同步采样精度

ADC1/ADC2同步触发，jitter<1ns：
对于1MHz信号，相位误差：
$$\Delta\phi = 2\pi \times 1\text{MHz} \times 1\text{ns} = 6.28 \times 10^{-3} \text{ rad} = 0.36°$$

幅度误差(cos(0.36°)≈1)可忽略 ✅

但ADC采样率3.6MSPS限制最高信号频率360kHz ❌

### 5. 优缺点

| 优点 | 缺点 |
|------|------|
| 单MCU方案理论上BOM最简单 | **DAC速度不足，无法产生>125kHz正弦波** ❌ |
| 双ADC同步采样方便 | ADC采样率仅3.6MSPS，无法处理1MHz信号 |
| FPU加速IIR实时滤波计算 | 100Hz-1MHz扫频需要10000点，耗时长 |
| 已知电路调试方便 | 需要外置DDS芯片，实际上不是纯MCU方案 |
| — | 1MHz信号实时IIR输出无法实现 |

**结论：方案A纯MCU不可行，仅作为理论对比。实际应以外置DDS+VGA+高速ADC为核心。**

#### 2.2 ADC采样链

| 器件 | 参数 |
|------|------|
| ADS8320 | 16-bit, 100kSPS (SPI), INL=±1.5LSB |
| PGA113 | 可编程增益(1~200), BW=10MHz @ G=1 |
| MAX4736 | 双SPST模拟开关, 选通Uin/Uout |
| OPA365 | 缓冲驱动, BW=50MHz |

**采样率选择**：
- 最高信号频率: 1MHz
- 采样率: fs = 100kSPS (ADS8320最大值)
- 奈奎斯特定理: fs/2 = 50kHz < 1MHz ❌

**问题**：100kSPS采样率远不能满足1MHz信号采样需求。

**修正方案**：使用**欠采样/等效时间采样**：
- 信号是周期重复的正弦波
- ADC在多个周期内分时采样
- 等效采样率可达 100kSPS × 10次重复 = 1MSPS

或者使用更高采样率ADC：
- 替换为ADS830E (8-bit, 60MSPS) 或 ADS4226 (12-bit, 125MSPS)
- 成本上升但可行

**修正后ADC方案**：

| 器件 | 参数 | 说明 |
|------|------|------|
| ADS4226 (或ADS4222) | 12-bit, 100MSPS, 双通道 | 同时采样Uin+Uout |
| THS4521 | 差分驱动 | 单端转差分驱动ADC |
| OPA847 | 缓冲 | G=+1, BW=3.9GHz |

**抗混叠LPF**：3阶Butterworth fc=2MHz (fs=100MSPS, fnyq=50MHz, 安全裕度大)。

#### 2.3 已知模型电路

MFB拓扑实现H(s)=5/(10⁻⁸s²+3×10⁻⁴s+1)：

```
         R1       C1
Vin ──/\/\/──┬───||──┬───  OpAmp(-)
             │       │       │
            R2       │       │
             │      GND      │
            /\/\──┬──        │
             │    │         │
            R3    │         │
             │   C2         │
            GND   │         │
                 GND        │
                            │
                     ┌──────┘
                     │
                    Vout
```

MFB传递函数：
$$\frac{V_{out}}{V_{in}} = \frac{-R_2/R_1}{s^2R_2R_3C_1C_2 + sC_1(R_2R_3/R_1 + R_2 + R_3) + 1}$$

令 R1=2k, R2=10k, R3=4.7k, C1=10nF, C2=2.2nF:

验证：
$$\omega_n^2 = \frac{1}{R_2R_3C_1C_2} = \frac{1}{10k \times 4.7k \times 10n \times 2.2n} = \frac{1}{1.034 \times 10^{-9}} = 9.67 \times 10^8$$
$$\omega_n = 3.11 \times 10^4 \text{ rad/s} \neq 10^4$$

需要迭代调整元件值或使用电位器微调。

**简化方案**：使用两级RC低通+同相放大(增益5)近似：
- RC1: R=15k, C=10nF, fc≈1.06kHz
- RC2: R=15k, C=10nF, fc≈1.06kHz
- 放大器: 同相放大G=5 (R3=10k, R4=2.5k)
- 总传递函数近似目标，误差可通过软件校正

### 3. 后处理与算法

#### 3.1 扫频测量（优化策略）

```
对数扫频: 100Hz ~ 1MHz, 每倍频程20点
总频点: 20 × log2(1M/100) = 20 × 13.3 ≈ 266点

for f in freq_list:
  AD9834.set_freq(f)
  AD8367.set_gain(根据频段预估算)
  delay(1/f × 10) // 10个周期建立时间
  
  // 等效时间采样 (对于>50kHz信号)
  if f > 50kHz:
    for i = 0 to 255:
      ADC_trigger(phase = i × 360/256)
      // 每周期采1点, 256周期采完
      buf[i] = ADS8320.read()
  else:
    // 直接采样256点 @ fs = 10×f
    for i = 0 to 255:
      buf[i] = ADS8320.read()
  
  Uin_rms = rms(buf_half1) // 前半是Uin
  Uout_rms = rms(buf_half2) // 后半是Uout
  mag_dB = 20log10(Uout_rms/Uin_rms)
  phase = correlation_phase(Uin_buf, Uout_buf)
  store(f, mag_dB, phase)

扫频时间估算:
  低频段(100-1kHz, 60点): 60 × 10/100Hz = 6s
  中频段(1k-100kHz, 80点): 80 × 256/f + 建立 = ~3s
  高频段(100k-1MHz, 126点): 126 × 256/f + 触发 = ~2s
  总时间 ≈ 11s << 2min ✅
```

#### 3.2 系统辨识算法

Bode图 → 传递函数参数：

**算法：最小二乘拟合**

测量向量: G_meas(ω_k) = |H_meas(jω_k)| (k=1~N)
模型: G_model(ω_k, p) = K / sqrt((1-(ω_k/ω_n)²)² + (2ζω_k/ω_n)²)

目标函数:
$$J(p) = \sum_{k=1}^{N} w_k [G_{meas}(\omega_k) - G_{model}(\omega_k, p)]^2$$

p = [K, ω_n, ζ] 三个参数迭代求解：
1. 初始值: K=G(ω→0), ω_n=ω_3dB, ζ=0.7
2. Gauss-Newton迭代: 3-5次
3. 检查残差: 若残差大 → 可能不是二阶系统

**Levy法离散实现**：
对传递函数离散化为差分方程，通过输入输出数据直接估计差分方程系数。

#### 3.3 IIR数字滤波器实时实现

辨识得H(s)后，双线性变换为IIR滤波器：

$$H(s) = \frac{K}{s^2/\omega_n^2 + 2\zeta s/\omega_n + 1}$$

采样率fs选择：最大信号频率1MHz, fs = 10MSPS (需外置高速ADC)

使用ADS4226 100MSPS, 抽取到10MSPS → MCU处理IIR:

二阶IIR差分方程：
$$y[n] = b_0 x[n] + b_1 x[n-1] + b_2 x[n-2] - a_1 y[n-1] - a_2 y[n-2]$$

STM32F407 @ 168MHz, 每个IIR运算约50个指令周期：
- 每采样点处理时间: 50/168M = 298ns
- 10MSPS: 每采样点100ns < 298ns ❌

**优化**：使用STM32F407浮点运算，单周期FMA指令：
- 每IIR节点: 3×乘加 + 2×乘 + 延迟更新 ≈ 20周期
- 20/168M = 119ns ≈ 100ns ✅（临界）

或降抽取率到5MSPS: 200ns >> 119ns ✅

### 4. 关键参数验证

#### 4.1 AD9834频率精度

28-bit频率寄存器, MCLK=25MHz:

$$f_{out} = \frac{FREQREG \times 25\text{MHz}}{2^{28}}$$

步进分辨率: 25MHz/2^28 = 0.093Hz
在1MHz时: FREQREG = 10^6 × 2^28 / 25×10^6 = 10737418

实际输出: 10737418 × 25MHz / 2^28 = 1,000,000.0Hz
误差: < 0.1Hz << 5% × 1MHz = 50kHz ✅

#### 4.2 VGA幅度控制精度

AD8367增益范围: -2.5dB ~ -42.5dB (8-bit SPI, 0.5dB步进)

需要输出2Vpp @ 已知电路输出端，计算所需VGA设定：

H(s)直流增益=5, 故Vin_need = 2Vpp/5 = 0.4Vpp

AD9834输出0.6Vpp → AD8367需要衰减:
$$G_{need} = 20\log_{10}(0.4/0.6) = -3.52\text{dB}$$

AD8367步进0.5dB，设定-3.5dB:
实际输出 = 0.6 × 10^(-3.5/20) = 0.6 × 0.668 = 0.401Vpp
误差: |0.401-0.4|/0.4 = 0.25% << 5% ✅

#### 4.3 ADC采样精度

ADS8320 (16-bit, Vref=2.5V):
- LSB = 2.5V/2^16 = 38.1μV
- INL = ±1.5LSB = ±57.2μV

在1MHz信号、满量程1Vpp输入时：
- SNR理论: 6.02×16+1.76 = 98.1dB
- 实际SNR @ 100kSPS ≈ 85dB
- ENOB ≈ (85-1.76)/6.02 = 13.8-bit

幅度测量精度:
$$\delta|H| = \frac{\sqrt{2} \times \text{LSB}}{V_{in,rms}} \times |H|$$

V_in,rms=0.353V, |H|=1 (0dB):
$$\delta|H| = \frac{1.414 \times 38.1\mu\text{V}}{0.353V} = 0.015\% \ll 10\% ✅$$

#### 4.4 等效时间采样相位精度

256次触发, 相位步进360°/256=1.406°:

对于1MHz信号, 时间步进:
$$\Delta t = \frac{1}{1\text{MHz}} \times \frac{1}{256} = 3.9\text{ns}$$

ADC触发抖动需要 << 3.9ns → 使用TIM定时器硬件触发:

STM32F407 TIM输出比较jitter < 1ns:
$$\Delta t_{jitter} = 1\text{ns} \ll 3.9\text{ns} ✅$$

```
FSMC → AXI → 更新IIR系数 (MCU辨识完成后)
```

IIR流水线结构(FPGA, 4级级联):
```
x[n] → Biquad1 → Biquad2 → Biquad3 → Biquad4 → y[n]
         ↓          ↓          ↓         ↓
       (b0,b1,b2,a1,a2) 系数来自MCU写入
```

每Biquad延迟: 4个时钟 @ 100MHz = 40ns
4级级联: 160ns << 1/1MHz = 1μs ✅

#### 3.4 系统辨识 (MCU侧)

FPGA提供高精度Bode数据后，MCU执行：

1. **滤波器类型识别**：从Bode图趋势判断低通/高通/带通/带阻
2. **传递函数拟合**：

对于二阶RLC网络：
- 低通: H(s) = 1/(LCs² + RCs + 1)
- 高通: H(s) = LCs²/(LCs² + RCs + 1)
- 带通: H(s) = RCs/(LCs² + RCs + 1)

从Bode图提取特征：
- 直流增益: H(ω→0) → 确定类型
- 谐振频率: 峰值/谷值位置 → ω_n
- 峰值高度: → ζ
- 滚降斜率: 确定阶数

**参数反推RLC值**：
$$\omega_n = \frac{1}{\sqrt{LC}}, \zeta = \frac{R}{2} \sqrt{\frac{C}{L}}$$

对于低通:
$$C = \frac{1}{2\zeta R \omega_n}, L = \frac{1}{\omega_n^2 C}$$

### 4. 关键参数验证

#### 4.1 DDS频率精度

FPGA 32-bit NCO @ 100MHz:
- 分辨率: 100MHz/2^32 = 0.023Hz
- 1MHz时FCW = 42949673
- 实际频率: 42949673 × 100M/2^32 = 999,999.98Hz
- 误差: 0.02Hz << 5% × 1MHz ✅

输出相位噪声:
- NCO量化本底: -6.02×32 - 1.76 = -194dBc/Hz
- DAC量化: -6.02×14 - 1.76 = -86dBc/Hz
- 整体相噪受限DAC ≈ -85dBc/Hz @ 10kHz ✅

#### 4.2 ADC采样信噪比

ADS4226双通道 @ 100MSPS, fin=1MHz:
- SNR = 70dB
- 信号带宽内噪声: -70dBc

双通道同步采样:
- Uin和Uout在同一芯片内采样
- 通道间匹配: ±0.05dB增益误差, ±0.1°相位误差
- 经校准后: 增益误差<0.01dB, 相位误差<0.02° ✅

#### 4.3 扫频辨识时间

FPGA硬件加速扫频:

| 阶段 | 时间 | 说明 |
|:----:|:----:|------|
| 266频点扫描 | 266×(10.24μs+2μs) ≈ 3.3ms | FPGA流水线处理 |
| 数据传输 | 266×12字节 ≈ 3.2KB → FSMC<0.1ms | AXI从FSMC读取 |
| 系统辨识 | 5次迭代 × 2ms = 10ms | Levy法/Gauss-Newton |
| IIR系数写入 | 5个系数 × 4级 = 20个32-bit写 < 1μs | FSMC写FPGA寄存器 |
| **总时间** | **< 15ms** ≪ 2min ✅ |

#### 4.4 IIR实时处理延迟

FPGA IIR流水线:
- ADC采样→ISERDES→IIR→DAC输出
- 延迟 = ISERDES(3clk) + FIFO(1clk) + IIR×4(16clk) + DAC延迟(5clk)
- = 25clk @ 100MHz = 250ns

对于1MHz信号, 250ns相当于0.09个周期:
- 群延迟误差: 250ns / 1μs × 360° = 90° ❌

**问题**：IIR滤波引入90°相位偏移, 但这是滤波器本身的物理特性(与模拟滤波器一致)。输入输出波形形态匹配, 相位偏移不影响"相同的输出波形"要求。

波形匹配验证（幅度）：
$$\text{误差} = \frac{|y_{sim} - y_{IIR}|}{|y_{sim}|} \times 100\%$$

FPGA 14-bit DAC重建:
- LSB = 3.3V/2^14 = 201μV
- 输出2Vpp时: 分辨率 = 201μV/2V = 0.01% << 10% ✅

### 5. 优缺点

| 优点 | 缺点 |
|------|------|
| FPGA NCO+DAC频率精度极高(0.023Hz) | 开发复杂度最高(Verilog+VHDL+ARM) |
| 双通道100MSPS ADC同时采集Uin/Uout | 需要高速DAC(AD9767)和高速ADC(ADS4226) |
| FPGA IIR流水线实时处理, 延迟250ns | BOM成本较高(~¥560) |
| 扫频时间<5ms(硬件加速) | 需要差分等长布线(LVDS @ 100MHz) |
| FFT硬件加速(1024点/10μs) | XC7A35T BGA焊接困难 |
| 系统辨识在MCU侧, 灵活可调 | 需要更多电源轨(1.0V/1.8V/3.3V/5V) |
| 可扩展为网络分析仪 | FPGA配置Flash需额外编程 |

## 三方案对比总结

| 维度 | 方案A (纯MCU) | 方案B (MCU+专用芯片) | 方案C (FPGA异构) |
|------|:---:|:---:|:---:|
| **BOM成本** | ~¥252(缺DDS) | ~¥380(含高速ADC) | ~¥570 |
| **可行性** | ❌ DAC/ADC均不足 | ✅ 需ADS4226修正 | ✅ 最优 |
| **DDS性能** | DAC 1MSPS: 最高~62kHz | AD9834: 37.5MHz ✅ | NCO+DAC: 20MHz ✅ |
| **ADC采样** | 3.6MSPS: 不足 | 100MSPS: 够用 | 100MSPS: 够用 |
| **扫频时间** | 太长(>60s) | ~11s | **<5ms** 🏆 |
| **系统辨识** | MCU, <1s | MCU, <0.5s | FPGA+MCU, <15ms |
| **IIR实时性** | 无法满足1MHz | 临界(5MSPS) | ✅ 无延迟 |
| **开发难度** | 低 | 中 | **高** |
| **推荐场景** | ❌ 不推荐 | ✅ 性价比之选 | 🏆 性能顶配 |

**推荐选择**：
- **方案B**为性价比最优解（将ADS8320替换为ADS4226后完全满足需求）
- **方案C**为性能最优解（FPGA加速扫频+IIR流水线，适合追求满分的团队）
- **方案A纯MCU不可行**，仅作为教学对比
