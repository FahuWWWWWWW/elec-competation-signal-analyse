# H题 信号分离装置 — 硬件方案分析

## 方案A：纯MCU方案（STM32H743VGT6 + 内置ADC/DAC + 软件I/Q解调）

### 1. 系统架构

```
[混合信号C输入] ──→ [缓冲跟随] ──→ [抗混叠LPF] ──→ [ADC内置]
  A+B (20~100kHz)    OPA2350         fc=200kHz       2路12bit
                                                      同步1MSPS
                                                         │
                                                         v
                                                  [STM32H743VGT6 (480MHz)]
                                                  ┌──────────────────────┐
                                                  │  采样缓冲 (DMA)       │
                                                  │                      │
                                                  │  Goertzel频率扫描     │
                                                  │  (20~100kHz@1kHz步进) │
                                                  │                      │
                                                  │  I/Q相干解调          │
                                                  │  ×cos → LPF → I_A   │
                                                  │  ×sin → LPF → Q_A   │
                                                  │  ×cos → LPF → I_B   │
                                                  │  ×sin → LPF → Q_B   │
                                                  │                      │
                                                  │  谐波分析(三角波检测)  │
                                                  │  波形重建(Sine/Tri)   │
                                                  └──────────┬───────────┘
                                                             │
                                                  ┌──────────v───────────┐
                                                  │  内置DAC (12bit×2)   │
                                                  │  150kSPS/ch           │
                                                  │  A'(t), B'(t)输出    │
                                                  └──────────────────────┘
```

### 2. 前端调理电路

**(a) 缓冲跟随器**
- **器件**: OPA2350 (双路, GBW=38MHz, SR=25V/μs)
- **配置**: 单位增益缓冲器 (电压跟随器)
- **输入阻抗**: >10MΩ (不加载加法器输出)
- **带宽**: 38MHz (>100kHz×100倍余量)

**(b) 抗混叠低通滤波器**
- **拓扑**: 3阶Butterworth, Sallen-Key
- **截止频率**: fc = 200kHz (100kHz最高信号频率×2倍)
- **计算**: R=8.2kΩ, C=97pF (取C=100pF)
  - fc = 1/(2π×8.2k×100p) = **194kHz** ✅
- **带外衰减**: -60dB/dec @ >200kHz
- **ADC采样率**: fs = 1MSPS (5×过采样)
  - Nyquist要求: fs > 2×100kHz = 200kHz ✅

**(c) ADC采样 (内置)**
- **ADC**: H743内置ADC1/ADC2 (12bit, 双ADC同步)
- **采样率**: 1MSPS (单通道)
  - 双ADC模式下同步采样: 只用ADC1(或SCAND模式)
  - 100kHz信号每周期10个采样点 → 充足
- **触发**: TIM2 PWM 1MHz触发
- **DMA**: 双缓冲(BufferA/B 各2048×16bit)
- **参考**: 内部VREF+ (3.3V)
- **过采样**: 硬件4×过采样 → 13bit等效 (H743支持)

**(d) DAC输出**
- **DAC**: H743内置DAC1/DAC2 (12bit, 双通道)
- **更新率**: 150kSPS/ch (通过TIM+DMA)
- **输出范围**: 0~3.3V
- **外部重建滤波器**: 2阶Butterworth LPF, fc=150kHz
  - 运放: OPA2350的另一路
  - R=10kΩ, C=106pF → fc≈150kHz
  - 消除DAC阶梯输出中的采样谐波
- **输出幅度调整**: 通过DAC数据乘系数, 确保峰峰值≥1V

### 3. 后处理与算法

**ADC配置**:
- 单通道, 1MSPS, 12bit
- DMA循环写入2048点缓冲区
- 每采集满2048点触发一次中断 → 处理一帧数据
- 帧时间: 2048/1M = **2.048ms** (满足≤20s分离时间要求)

**频率检测 (Goertzel扫描)**:
```
for f = 20000 to 100000 step 1000:   // 81个候选频率
    power = goertzel(buffer, f, fs)   // Goertzel算法
    // Goertzel: v_k[n] = x[n] + 2·cos(2πk/N)·v_k[n-1] - v_k[n-2]
    // power = v_k[N-1]² + v_k[N-2]² - 2·cos(2πk/N)·v_k[N-1]·v_k[N-2]
find 2 max peaks (min separation = 5kHz)
→ fA, fB
```
- 81次Goertzel × 2048点 = 166k次MAC运算
- H743@480MHz: 约3ms完成 ✅ (远超≤20s要求)

**I/Q相干解调**:
```
对每个检测到的频率 fA, fB:
    // 混频
    I_A[n] = C[n] × cos(2π×fA×n/fs)
    Q_A[n] = C[n] × sin(2π×fA×n/fs)
    
    // 低通滤波 (CIC或FIR)
    lpf_cutoff = 2kHz (保留包络信息, 滤除2fA分量)
    I_A_lpf = FIR_filter(I_A, h_lpf)
    Q_A_lpf = FIR_filter(Q_A, h_lpf)
    
    // 幅度与相位
    Amp_A = 2 × sqrt(mean(I_A_lpf²) + mean(Q_A_lpf²))
    Phase_A = atan2(mean(Q_A_lpf), mean(I_A_lpf))
```

**谐波分析 (波形类型检测)**:
```
// 对A和B分别检测谐波含量
harm_3rd_A = goertzel(buffer, 3×fA, fs)
harm_5th_A = goertzel(buffer, 5×fA, fs)
ratio_A = (harm_3rd_A + harm_5th_A) / power_A

if ratio_A > 0.05: type_A = TRIANGLE
else: type_A = SINE
```

**波形重建**:
```
// 正弦波重建
A'[n] = Amp_A × sin(2π×fA×n/fs + Phase_A)
B'[n] = Amp_B × sin(2π×fB×n/fs + Phase_B)

// 三角波重建 (用查找表, 只有基波幅度和相位相同)
// 三角波查找表: triangle_table[1024] = {-1.0 ~ 1.0}
// 归一化后三角波基波幅度 = 8/π² ≈ 0.81, 需放大
A'[n] = Amp_A × (π²/8) × triangle_lut(phase_A[n])
```

**DAC输出**:
- TIM+DMA循环输出A'和B'的波形数据
- 输出更新率: 150kSPS (每100kHz信号每周期1.5点 → 略少)
- 改进: 使用DAC的三角波/方波模式下用定时器更新
- 或扩频: 用DMA乒乓缓冲提高更新率到300kSPS

### 4. 关键参数验证

**(1) Goertzel频率分辨率**
- N=2048, fs=1MSPS
- Δf_bin = fs/N = 1M/2048 = **488.3Hz**
- 频率扫描步进: 1kHz = 约2个bin
- 可分辨最小频率间隔: 5kHz (题目要求, fA和fB间隔≥5kHz)
- 实际检测能力: 5kHz / 488Hz ≈ 10.2个bin → 完全可区分 ✅
- 对于5kHz整数倍的信号(20kHz, 25kHz, 30kHz...):
  - 最大误差: ±500Hz (扫描步进的一半)
  - 相对误差@20kHz: 500/20000 = **2.5%** ✅

**(2) I/Q解调幅度误差**
- 幅度提取: Amp = 2 × √(I² + Q²)
- ADC 12bit + 4×过采样 → 13bit有效 → 8192级
- 量化噪声: V_q = 3.3V/8192 = **0.403mV**
- 输入信号: 1Vpp = 2Vpp(差分思维简化为单端1Vpp)
  - 实际ADC输入峰峰值: 1V(经耦合)
  - SNR = 20log10(1/√2 / 0.000403) = **64.9dB**
- 幅度误差: ≈ 1/√(SNR_linear) = 1/1757 ≈ **0.057%**
- 指标要求: <0.5% ✅ (远超)

**(3) DAC输出幅度精度**
- 12bit DAC: 3.3V/4096 = **0.806mV**每级
- 目标输出: 1Vpp → 需要1241LSB
- 设置误差: ±0.5LSB → 幅度误差 = 0.5/1241 = **0.04%**
- 但DAC输出需经LPF, 有插入损耗(~5%) → 需软件预补偿
- 最终输出幅度误差: <0.5% ✅ (经过校准)

### 5. 优缺点

| 优点 | 缺点 |
|------|------|
| BOM极低(¥147), 纯MCU方案 | 内置DAC更新率150kSPS略紧 |
| H743双精度FPU加速I/Q计算 | 三角波重建用查找表, 精度受限于点数 |
| Goertzel扫描+IQ解调全部软件实现 | DAC输出需外接重建LPF(运放) |
| 12bit ADC+过采样=64dB SNR | 频率扫描步进1kHz, 不能精确对准5kHz整数倍 |
| 开发简单, 无需外部器件 | 输出通道数受限(内置2路DAC, 刚好够A'/B') |

## 方案B：MCU+专用芯片方案（STM32H723ZGT6 + AD7606 + 双DAC）

### 1. 系统架构

```
[混合信号C输入] ──→ [缓充+LPF] ──→ [ADC AD7606] ←── [CONVST (TIM触发)]
                        fc=200kHz      16bit, 200kSPS
                                           │ SPI
                                           v
                                    [STM32H723ZGT6 (550MHz)]
                                    ┌──────────────────────┐
                                    │ Goertzel频率扫描      │
                                    │ (利用硬件CORDIC加速)  │
                                    │                      │
                                    │ I/Q解调 (ARM DSP库)   │
                                    │ 谐波分析              │
                                    │ 波形重建              │
                                    └─────┬─────┬──────────┘
                                          │     │
                                     [SPI DAC] [SPI DAC]
                                     AD5760    AD5760
                                       │         │
                                       v         v
                                    [LPF]     [LPF]
                                     fc=150kHz  fc=150kHz
                                       │         │
                                       v         v
                                   A'输出     B'输出
```

### 2. 前端调理电路

**(a) 缓冲+抗混叠**
- **器件**: OPA2350 (跟随器 + 2阶LPF)
- **截止频率**: fc = 200kHz
- 同方案A

**(b) 专用ADC — AD7606**
- **分辨率**: 16bit (无失码)
- **通道数**: 8通道同时采样 (只用1通道)
- **采样率**: 200kSPS
- **输入范围**: ±10V / ±5V (软件选择±5V)
- **内置特性**: 1MΩ输入阻抗, 自带抗混叠(fc=22kHz)
- **触发**: CONVST引脚由STM32 TIM输出200kHz PWM触发
- **数据**: SPI模式读取 (16bit, 每转换完成触发BUSY中断)
- **SNR**: 89dB (理论) / 85dB (实测)

**注意事项**: AD7606内置抗混叠fc=22kHz, 不足够保护100kHz信号
- 外部追加2阶LPF, fc=200kHz (使用OPA2350)
- 或: 使用采样率200kSPS, 对100kHz信号每周期仅2个采样点
  - Nyquist勉强满足: fs=200kHz > 2×100kHz=200kHz → 临界
  - 实际需要: 适当提高fs或增加通道交织

**(c) 双DAC输出 — AD5760**
- **器件**: AD5760 (16bit, 单通道, SPI接口)
- **更新率**: 1MSPS (远超需求)
- **输出范围**: ±10V / ±5V (可设)
- **建立时间**: 2.5μs (到0.0015%)
- **THD**: -120dB (典型)
- **使用**: 两个独立AD5760分别输出A'和B'波形
- **控制**: 通过SPI2/SPI3各控制一个DAC
- **时钟**: TIM触发DMA, SPI传输16bit数据, 更新率200kSPS

### 3. 后处理与算法

**ADC配置**:
- AD7606: 200kSPS, 单通道, SPI模式
- CONVST: 200kHz PWM (定时器TIM3_CH1)
- BUSY → EXTI中断 → SPI读取转换结果
- DMA: 将SPI数据直接传送到SRAM (2048点循环缓冲)
- 帧: 2048点 → 帧时间 = 2048/200k = 10.24ms

**注意事项**: 200kSPS对100kHz只有2点/周期, 奈奎斯特临界。
- Goertzel算法在采样率刚好2倍信号频率时性能下降
- 需要: 1) 使用抗混叠保护; 2) 增加采样窗长度(N=4096提升频率分辨率)
- 或: 使用AD7606的双通道模式, 以相位偏移90°获得交织采样(等效400kSPS)

**Goertzel扫描 (硬件CORDIC加速)**:
```
// STM32H723内置硬件CORDIC可加速三角函数计算
// Goertzel系数 cos(2πk/N) 可用CORDIC硬件计算
// 加速比: 软件atan2≈200周期 → 硬件CORDIC≈7周期

// 扫描范围: 20~100kHz @ 1kHz步进 (81个频率)
// 每个频率2048点Goertzel: 81 × 2048 = 166k次MAC
// H723 FPU+SIMD: 约2ms完成 ✅
```

**I/Q解调**:
- 使用ARM DSP库: arm_fir_f32 (FIR滤波器)
- 滤波器系数: 预先计算, 存入Flash
- 低通滤波: 截止2kHz, 阶数64

**波形重建**:
- 通过SPI DMA刷新AD5760
- 波形数据预生成, 存入SRAM
- 三角波: 用CORDIC计算相位 → 查三角波表
- 输出更新率: 200kSPS (每周期2点→对100kHz)
  - 用DAC内置的线性插值功能

### 4. 关键参数验证

**(1) 16bit ADC动态范围**
- AD7606: 16bit, ±5V → LSB = 10V/65536 = **152.6μV**
- 输入信号: 1Vpp = 0.3535Vrms (正弦)
- 信号能量: SNR = 20log10(0.3535/0.000153) = 67.3dB
- 但AD7606实际SNR=85dB → 量化噪声非主导
- 等效ENOB: ≈ 13.8bit (受前端噪声限制)
- 幅度测量误差: ≈ 0.05% ✅ (0.5%要求)

**(2) Goertzel频率扫描精度**
- N=4096, fs=200kSPS → Δf = 200k/4096 = **48.8Hz**
- 扫描步进: 1kHz (每20.5个bin)
- 频率检测误差: ±24.4Hz (半个bin)
- 相对误差@20kHz: 24.4/20000 = **0.12%** ✅
- 频率分离能力: 可区分间隔≥100Hz的两个信号 ✅ (5kHz要求远超)

**(3) 总谐波失真对三角波检测的影响**
- 检测3次谐波: Goertzel @ 3×fA
- 三角波3次谐波幅度: 基波的1/9 = 11%
- ADC量化噪声: -85dB relative → 远小于11%
- 谐波检测阈值: 设为5% → 可清楚区分正弦/三角波 ✅

### 5. 优缺点

## 方案C：FPGA异构方案（Artix-7 XC7A35T + STM32F407VGT6）

### 1. 系统架构

```
[混合信号C输入] ──→ [缓冲+LPF] ──→ [ADC AD7606] ──→ [XC7A35T (Artix-7)]
                        fc=200kHz      16bit, 200kSPS   ┌───────────────────────┐
                                                        │ ADC接口 + FIFO缓存    │
                                                        │                       │
                                                        │ Goertzel加速器 (并行)  │
                                                        │ 81个频率同时计算       │
                                                        │ (DSP48E1×4, 流水线)   │
                                                        │                       │
                                                        │ I/Q解调器 (并行)       │
                                                        │ 2通道×2路混频+FIR     │
                                                        │                       │
                                                        │ 波形发生器 (DDS)      │
                                                        │ 正弦/三角查找表        │
                                                        │ 16bit/通道            │
                                                        └──┬──────┬────────────┘
                                                           │      │
                                                   [DAC AD5760] [DAC AD5760]
                                                        │      │
                                                       [LPF]  [LPF]
                                                        │      │
                                                       A'     B'
                                                       
                                              [STM32F407VGT6 (168MHz)]
                                              ┌────────────────────────┐
                                              │ 控制/配置/UI           │
                                              │ SPI配置FPGA参数        │
                                              │ 显示/按键/串口         │
                                              └────────────────────────┘
```

### 2. 前端调理电路

**(a) 缓冲+抗混叠**
- **器件**: OPA2350 (同方案A/B)
- 截止频率: fc=200kHz

**(b) ADC — AD7606**
- **器件**: AD7606 (16bit, 200kSPS, 8ch)
- 同方案B, 使用单通道
- **数据**: SPI或并行模式 (建议并行模式, 速度更快)
- 并行模式: 8/16bit总线 → FPGA通用IO直接读取
- 采样速率: 200kSPS, 每5μs一次16bit采样

**(c) FPGA内部处理引擎**

**Goertzel加速器 (关键模块)**:
```
结构: 81个并行Goertzel单元 (每个频率一个)
资源:
  每个单元: 2×DSP48E1 (复数MAC) + 1×BRAM (系数表)
  共: 162×DSP48E1 (超标! XC7A35T只有90个DSP48E1)

优化: 时分复用 → 4个DSP48E1分时计算81个频率
  每个频率轮询计算: 2048点/频率 × 81频率 = 166k次MAC
  4个DSP48E1并行: 166k/4 = 41.5k周期
  @80MHz: 41.5k/80M = 0.52ms ✅
```

**I/Q解调器**:
```
流水线结构:
  输入 → NCO(查表) → 混频器(乘法器) → CIC抽取 → FIR补偿
  2通道并行 (A和B同时解调)
  
  NCO: 2个32bit累加器, 产生cos/sin
  混频: 4×DSP48E1 (I_A, Q_A, I_B, Q_B)
  CIC: 3阶, 抽取R=4
  FIR: 31阶, 系数预存BRAM

  延迟: 约50个时钟周期
  吞吐: 每时钟1个采样点
```

**波形发生器**:
```
DDS引擎:
  相位累加器(32bit) ×2 (A和B)
  正弦查找表: 1024点×16bit (BRAM实现)
  三角波查找表: 1024点×16bit (BRAM实现)
  
  输出: 16bit并行 × 2通道
  → 通过SPI或并行总线输出到AD5760
```

### 3. 后处理与算法

**FPGA处理流程 (全流水线, 无MCU干预)**:

```
1. ADC采样: 200kSPS, 16bit → FPGA FIFO (深度4096)
2. 每帧(4096点)采集完成 → 触发Goertzel加速器
3. Goertzel扫描(81频率) → 返回功率向量[81] → 峰值搜索
4. 找到fA, fB → 配置NCO产生对应频率的cos/sin
5. I/Q解调 → 实时流水线: ADC数据→混频→CIC→FIR
6. 提取幅度Amp_A, Amp_B, 相位Phase_A, Phase_B
7. 谐波分析 → 判断正弦/三角波
8. 波形发生器 → 重建A'和B'
9. DAC输出: 通过并行口送到AD5760

总延迟: < 1ms (从ADC输入到DAC输出)
```

**STM32控制**:
- 初始化: 配置FPGA (SPI写入控制寄存器)
- 监控: 读取FPGA状态寄存器
- 显示: LCD显示当前频率/幅度/波形类型
- 相位控制(发挥部分): 通过SPI写入相位偏移值

### 4. 关键参数验证

**(1) FPGA Goertzel加速比**
- MCU(H743): 81频率 × 2048点 × 2MAC = 332k次MAC @ 480MHz → 约3ms
- FPGA: 4个DSP48E1并行, 每周期4次MAC @ 80MHz
  - 计算时间 = 332k/4/80M = **1.04ms**
  - 加上流水线延迟: 约1.5ms
- 加速比: 3ms/1.5ms = **2倍** (主要是MCU已很快)
- 但FPGA优势在: 全流水线连续处理, 不占用MCU时间

**(2) 幅值误差分析**
- AD7606: 16bit, LSB=152.6μV
- 信号1Vpp → 0.3535Vrms → 2317LSB
- I/Q解调后幅度: SNR = 85dB (ADC) + 处理增益
  - CIC抽取R=4: 处理增益 = 10log10(R) = 6dB
  - 总SNR: 85+6 = 91dB
- 幅度相对误差: ≈ 1/√(10^(91/10)) = 1/35481 = **0.0028%**
- 实际前端噪声限制: ≈ 0.05% ✅ (远超0.5%要求)

**(3) 相位控制精度 (发挥部分)**
- 相位累加器: 32bit → 相位分辨率 = 360°/2³² = **8.4×10⁻⁸度**
- 查找表: 1024点 → 相位分辨率 = 360°/1024 = **0.35°**
- 实际: 使用线性插值 → 有效分辨率≈0.01°
- 指标要求: 分辨率5°, 误差≤5° ✅

**(4) 输出更新率**
- AD5760最大更新率: 1MSPS
- FPGA每5μs发送一次16bit并行数据
- 100kHz信号: 每周期10个输出点 → 波形平滑
- 输出LPF fc=150kHz, 消除阶梯 ✅

### 5. 优缺点

| 优点 | 缺点 |
|------|------|
| FPGA全流水线处理延迟<1ms | BOM成本中高(¥500+) |
| 81频率Goertzel并行扫描, 秒级完成 | XC7A35T的DSP资源有限(90个), 设计需优化 |
| AD7606 16bit+FPGA处理=0.05%幅度精度 | 需要并行ADC接口, FPGA引脚要求高 |
| 双AD5760输出波形质量极高 | 开发周期长(Verilog时序约束复杂) |
| 可实现真正连续实时输出(无帧间隙) | 功耗~1.5W |
| 相位控制精度0.01°, 远超指标 | FPGA与MCU间的通信协议需自定义 |
